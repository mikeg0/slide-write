Slide Write — per-origin config

Enable the widget only on origins you trust. Set Shim URL to your shim's address — e.g. http://localhost:4040 (VS Code forwards it to your browser). Leave it blank only for the reverse-proxy fallback, where it derives <origin>/_slidewrite. The token is the primary security boundary — use a long random secret.

Image generation (Gemini “nano banana”)

One Gemini API key, shared across all origins, used to generate images for the 🖼️ image button. Get a key from Google AI Studio. Stored locally; never committed.

A project's image-save conventions (path, naming, resize, DB/CDN steps) belong in the target repo — its CLAUDE.md, or a .claude/skills/ image-asset skill (run the shim with --use-skills to load skills). The per-origin “Image steps (override)” field below only layers extra instructions on top, per developer.

Connection

How often the panel re-checks the shim (/health + /meta) while it's open, so the status flips as the shim goes up/down. Only polls while the panel is visible. Default 5 seconds; minimum 1.
